jQuery

Introduction

jQuery is a popular Javascript library for a whole range of page manipulations that are difficult or impossible with HTML or CSS. Usually we use it to create animations or alter the contents of a page without reloading it. Remember that these manipulations are only altering the page as it exists in the memory of the browser. If you reload the page, you’ll reset everything back to the way it was.

jQuery can involve up to three chunks of Javascript, and they need to be loaded via the <script> tags in the order described below:

  1. Take a look at how this HTML page is structured. Up in the <head> tag, I’ve first embedded the jQuery library itself — this is always necessary. Many people choose to load this file locally, just like a CSS file. But here I’ve been lazy and just loaded it straight from Google’s servers (<script src="http://ajax.googleapis.com/ajax/libs/jquery/1.6.1/jquery.js"></script>), who host it for the rest of the internet to use. This works fine, but if Google ever takes the file down, the functionality on this page would break. The library itself is really only comprehensible for the browser: you never need to look at it or edit it.
  2. Second we have a jQuery plugin: <script src="js/jquery.lightbox-0.5.js" type="text/javascript"></script>; this is a Javascript file I downloaded from the developer’s website and am loading locally. Plugins are only necessary for fancier jQuery effects: in our case it’s just the lightbox effect (Part 3) that requires it.
  3. Finally we always write jQuery code that is specific to our page and the effects we want. Often you will see this written in the HTML page between <script> tags. In this exercise, though I’ve put everything in a separate text file (some-scripts.js) and linked to it.

Credits: this section is adapted from the Simple Toggle with CSS and jQuery tutorial, by Soh Tanaka.

  2. Look at the top of some-scripts.js. For now, you can ignore the $(document).ready(function(){... line of code. It just tells jQuery to wait until the whole page has loaded before starting to do anything.
  3. Notice the comments: Comments in Javascript/jQuery begin with two slashes and don’t need to be closed (the whole line becomes inactive). Try commenting out the $(".toggle-content").hide(); line of code, saving the file, and reloading the web page. In addition, we use the toggleClass() function to dynamically add and remove the active class whenever we click on the h3. This is useful because we can write a rule for the .active CSS class in our exercise10.css file. Try opening that CSS file and finding the rule that modifies the link inside the h3 when/if it has the active class. Change the orange to green and test the page.
  6. Part 1 also makes use of the next() function: here it is cleverly used to dynamically target whatever HTML element immediately follows the h3 that was clicked. Go into exercise10.html, locate the HTML for Toggle 3, and insert something like <p>Test</p> right before <div class="toggle-content">. Save the file and reload the page. You should see that the paragraph you inserted has become the thing that gets shown/hidden when clicking, because it is now the element that immediately follows the <h3> trigger.

Part 2: In-page Tabs

Credits: this section is adapted from the Simple Tabs with CSS & jQuery tutorial, by Soh Tanaka.

  1. Try out the tabs below. Look at the HTML source, and add a fifth tab and corresponding content box. Notice how this structure differs from Part 1: the tab links do not directly preceide their corresponding content boxes. Instead, they are linked via hash links (<a href="#tab1">) to their content <div>s: this is nice logical markup: if Javascript were turned off, the content of all five boxes would pour down the page, but the "tab" links would still function as jump links, zooming the user down to the correct content area when they clicked a tab. This is the principle of graceful degradation in action: jQuery is being used to enhance the layout and present content in a cooler/more compact way, but the content is always accessible and logically marked up, even if accessed by an ancient browser, screen reader, etc.
  2. The cleverest part of jQuery is that it uses CSS selector syntax to target the HTML elements you want to manipulate. Go back to some-scripts.js and read through the jQuery code for Part 2, step by step, as you look at the HTML for Part 2 in exercise10.html. The functions may be new, but you should recognize the HTML selectors, including pseudo classes. (Note that :first is actually specific to jQuery — in CSS we only have :first-child). Try commenting out each line in turn, saving the file, and reloading the page. When you’re done, make sure everything is the way it was when you started.
  3. This example is particularly interesting because it makes use of Javascript variables (activeTab) and the fact that the tabs are linked to the IDs on the tab-content divs. Notice in particular how when you comment this line out, the tabs stop working.
  4. Using the jQuery code from Part 1 as a model, find where the slideToggle speed was set and try modifying Part 2 so that the initial showing of the first tab content happens at a fast speed instead of 800 milliseconds. Now try using that same syntax on the hide() function,which is used to hide all the tab content on click. Try making that same hiding function happen over 1000 milliseconds. Then try 100 milliseconds. Hint: words need quotation marks, numbers don’t.

Part 3: Lightbox

Credits: this section makes use of the jQuery Lightbox plugin by Leandro Vieira Pinho.

  1. Take a look at Part 3 in both somescripts.js and exercise10.html. With Set 1 (the Process Type Foundry images), I’ve only used the lightbox() function by itself. Try it out in the browser by clicking on the thumbnails. Now comment out this line and re-test the page. Notice that since the thumbnails are simply linked to their corresponding JPGs with plain HTML, the page still “works” (though the user needs to use the back button). Once again we have “graceful degradation”: if something goes wrong or the user doesn’t have Javascript turned on, our markup is good enough that the page does not fail completely.
  2. With Set 2, I’ve invoked the lightbox() function again, but this time I’ve used some of the custom variables that the plugin makes available. Try tinkering with these values. Remember that speeds are measured in milliseconds. Try tinkering with these values: change the background overlay color, its opacity, etc.
  3. Finally, visit the developer’s website, go to the “Extend” section, and pick another value to add to the code to change some other attribute of the lightbo.
